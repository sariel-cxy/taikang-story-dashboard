# 泰康故事库系统化建设 PRD v0.1：RD 对接与方案评估版

日期：2026-05-14

## 文档定位

当前已有泰康故事库静态页面，可作为展示原型和字段参考。集团品牌部的目标是建设可实时上传、编辑、审批、发布的故事库系统。正式系统需要支持线上编辑、审批、权限、内容可见范围控制、动态仪表盘、版本记录和操作日志等能力。

当前静态页面不能作为正式系统方案，只能作为原型参考。本 PRD v0.1 用于技术对接、方案评估和排期沟通，帮助 RD 评估平台承载能力、开发边界、接口依赖、双方分工和项目排期。待明确平台能力后，再形成可开发版 PRD v1.0。

## 1. 项目背景

目前已有泰康故事库静态展示页面，支持故事浏览、搜索、筛选、图表统计和故事卡查看。该页面主要用于阶段性汇报和原型展示，能够直观呈现故事库的内容结构、标签体系和仪表盘展示方式。

当前数据来自 Excel，通过脚本导出为前端静态数据文件，再由浏览器端页面读取并展示。当前系统没有后台、数据库、登录、权限、审批、发布、版本记录、审计日志等正式系统能力。

泰康希望将故事库升级为集团大内容平台内可运营的故事库系统，实现故事资产的统一管理、持续维护、审批发布、内容复用和动态统计分析。

## 2. 建设目标

### 2.1 业务目标

- 统一沉淀泰康集团故事资产，形成可持续维护的集团故事库。
- 支持故事持续更新、审核和复用，避免故事资产分散在 Excel、文档或个人文件中。
- 支持不同传播场景使用不同故事范围、字段范围或内容版本。
- 支持故事库实时或准实时统计分析，为选题、汇报和传播规划提供数据依据。
- 支持集团内部安全、合规、权限管理，避免敏感字段或未审内容外泄。

### 2.2 系统目标

- 内容在线维护。
- 审批流。
- 权限管理。
- 内容可见范围控制。
- 动态仪表盘。
- 版本记录和审计日志。
- 历史 Excel 数据导入。

- 无数据库。

- 无发布流。

## 4. 用户角色

| 角色 | 典型用户 | 核心权限 | 备注 |
|---|---|---|---|
| 故事编辑人 | 品牌部内容编辑、项目成员 | 新增故事、编辑草稿、维护基础字段、提交审核 | 不应直接发布 |
| 标签维护人 | 内容运营、资料整理人员 | 维护标签字段、补充分类、调整故事标签 | 可与编辑人重合 |
| 内容审核人 | 品牌负责人、内容审核负责人 | 查看待审核内容、审核通过、退回修改、填写审核意见 | 需保留审核记录 |
| 内容发布人 | 品牌部授权发布人员 | 发布、撤回、下架 | 需根据权限控制 |
| 业务查看人 | 业务部门、领导汇报使用者 | 查看授权范围内故事、筛选、搜索、查看仪表盘 | 不具备编辑和发布权限 |
| 系统管理员 | 平台管理员 | 用户权限、字段配置、标签配置、流程配置、系统维护 | 具体边界需 RD 确认 |

## 5. 核心业务流程

### 5.1 主流程

```text
故事录入/导入
  -> 编辑完善
  -> 提交审核
  -> 审核通过/退回
  -> 发布
  -> 授权范围内展示
  -> 仪表盘更新
  -> 后续生成新编辑版本 / 撤回 / 下架
```

### 5.2 状态流转表

| 状态 | 触发动作 | 下一状态 | 说明 |
|---|---|---|---|
| 草稿 | 新增、导入、保存 | 草稿 / 待审核 | 编辑中，未进入审核 |
| 待审核 | 提交审核 | 审核通过 / 退回修改 | 等待审核人处理 |
| 退回修改 | 审核退回 | 草稿 / 待审核 | 编辑人修改后可再次提交 |
| 审核通过 | 审核通过 | 待发布 | 内容具备发布条件 |
| 待发布 | 发布 | 已发布 | 等待发布人发布 |
| 已发布 | 发布成功 | 已发布 / 已撤回 / 已下架 | 已在授权范围内展示 |
| 已撤回 | 撤回发布 | 已撤回 | 从展示范围撤回，需保留撤回记录 |
| 已下架 | 下架 | 已下架 | 不再展示，需保留下架记录 |

### 5.3 已发布内容修改规则

- 已发布内容如需修改，应生成新编辑版本。
- 新编辑版本进入草稿/待审核流程。
- 原已发布版本继续保留，直到新版本审核通过并发布。
- 需支持撤回、下架和发布记录留痕。
- 版本替换、历史版本可见范围需 RD 根据平台能力确认。

## 6.0 一期 MVP 范围

一期 MVP 目标是先完成故事库从“静态展示原型”到“平台内可管理系统”的最小闭环，不要求一次性覆盖复杂多版本管理。

| MVP 能力 | 范围说明 |
|---|---|
| 历史 Excel 数据导入 | 支持现有故事数据批量导入，具备字段映射和基础校验 |
| 故事新增、编辑、保存、查看 | 支持平台内新增故事、编辑字段、保存草稿、查看详情 |
| 基础字段和标签维护 | 支持维护核心字段、标签和基础枚举值 |
| 角色权限 | 支持编辑、审核、发布、查看角色权限 |
| 基础审批流 | 支持提交审核、审核通过、退回修改、审批意见留痕 |
| 内部故事库展示 | 支持内部授权用户搜索、筛选、查看故事 |
| 基础状态和可见范围 | 支持内部可见、公开可用、待审、受限等基础状态或可见范围 |
| 基础动态/准动态仪表盘 | 支持基于正式数据统计核心指标 |
| 基础版本记录和操作日志 | 支持记录新增、编辑、审核、发布、撤回、下架等关键动作 |

## 7. 内容字段模型初稿

字段模型以当前静态原型和 Excel 字段为基础，正式字段类型、必填规则、可见范围、字段级权限和导入映射方式需 RD 根据平台能力确认。完整字段表见附件：《泰康故事库系统化建设 PRD v0.1 - 附件：内容字段模型初稿》。

### 7.1 字段分组摘要

| 字段分组 | 关键字段举例 | 说明 |
|---|---|---|
| 基础信息字段 | 编号、故事标题、故事摘要、首次发布时间、时间、地点 | 支撑列表、详情、搜索和基础统计 |
| 标签字段 | 一级标签、二级标签、三级标签、叙事类型、主角角色、价值观标签 | 支撑筛选、分类和仪表盘统计 |
| 内容详情字段 | 金句、背景、核心情节、关键转折/细节、结果/影响、价值意义、原文 | 支撑故事卡、汇报引用和内容复用 |
| 来源字段 | 来源标题、来源类型、来源链接、故事提供者、关键人物名称、备注 | 多数为内部或受限字段，需权限控制 |
| 权限/状态字段 | 内容状态、审批状态、发布状态、重要程度、是否适合对外传播、可见范围 | 支撑审批、发布和可见范围控制 |
| 审计字段 | 创建人、创建时间、最近修改人、审核人、版本号、操作日志 | 支撑追溯、审计和版本管理 |

### 7.2 关键字段说明

- `故事标题`、`故事摘要`、`一级标签（战略概念）`、`二级标签（业务板块）` 建议作为核心必填字段。
- `来源链接`、`故事提供者`、`备注`、`原文`、`关键人物名称` 等字段可能涉及内部信息或人物信息，建议默认按内部/受限字段处理。
- `是否适合对外传播`、`可见范围`、`发布状态` 是发布和字段展示控制的关键字段。
- `内容状态`、`审批状态`、`版本号`、`操作日志` 需与平台审批流、发布流和审计能力联动。
